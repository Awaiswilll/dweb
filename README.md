# dweb вҖ” P2P Self-Hosting OS

> **One install. Every stack. Your own internet.**

dweb is a **self-hosted P2P dev portal** that transforms any machine into a **personal cloud** вҖ” a decentralized node where you own your services, your domains, and your data. Built-in AI agents help you build, host, and publish any web architecture from your own machine, accessible to the world via P2P.

## Core Features

### рҹ–ҘпёҸ Service Management Dashboard
Start/stop services with one click, monitor CPU/memory/ports, view logs:
- **Static Sites** вҖ” Serve any HTML/CSS/JS folder
- **Node.js APIs** вҖ” Express, Fastify, and more
- **Python Web Apps** вҖ” Flask, FastAPI, Django
- **PHP Sites** вҖ” WordPress, Laravel, or plain PHP
- **File Browser** вҖ” Upload, manage, and share files through your browser
- **Custom Services** вҖ” Any port, any stack

### рҹҢҗ P2P Networking & Discovery
Every dweb installation is a **node** on a decentralized network:
- **Peer discovery** вҖ” Find other dweb nodes automatically via UDP multicast, file-based discovery, or relay-mediated signaling
- **Direct connections** вҖ” WebRTC encrypted P2P links with Google STUN pre-configured
- **Relay fallback** вҖ” WebSocket + HTTP polling for NAT traversal, with automatic reconnection and exponential backoff
- **Connection UI** вҖ” Dedicated Connect dialog with mode selection (Direct P2P / Via Relay), hover info tooltips explaining each connection type, and inline peer browser with search and filtering
- **Paginated peer lists** вҖ” Discovered peers and connected remotes displayed in pages of 15 with Prev/Next navigation and page counter
- **Network status bar** вҖ” Real-time relay status, peer count, online mode indicator, and descriptive hover tooltips
- **Multiple online modes** вҖ” Local Only, P2P Visible (discoverable), P2P Anonymous (connect out only)
- **Auto-registration** вҖ” Each instance auto-registers as a peer on startup for immediate discoverability
- **Persistent peer registry** вҖ” Peers survive server restarts via disk-backed storage (`/tmp/dweb-peers.json`)
- **Tor routing** вҖ” Optional Tor daemon integration with one-click toggle in the Network header
  - *Auto-detects* if Tor is installed, shows real-time status (running/installed/unavailable)
  - *One-click toggle* вҖ” Starts/stops Tor daemon (via `kalitorify` or `nohup tor`), server tracks state server-side
  - *Auto-anonymous* вҖ” Enabling Tor auto-switches instance to `p2p-anonymous` mode (Visible would defeat Tor's purpose)
  - *Status bar indicator* вҖ” Purple `Tor: Routing` badge with SOCKS5 proxy address on hover
  - *Verify endpoint* вҖ” `GET /api/tor/test` probes the SOCKS5 proxy port (127.0.0.1:9050) to confirm it's reachable
- **P2P File Transfer** вҖ” Share files directly between instances

### рҹҸ·пёҸ .dweb Domain System
Register and manage domains on the decentralized network:
- **Free tier** вҖ” 1 `.dweb` domain, basic P2P hosting
- **Premium tier** ($3/mo) вҖ” 5 domains, relay cache
- **Business tier** ($10/mo) вҖ” unlimited domains, cloud shift
- **Cross-peer domain resolution** вҖ” Resolve `.dweb` domains registered on any connected peer instance; queries all connected peers in parallel with 60s result caching
- **Auto domain assignment** вҖ” Publishing a project can optionally auto-assign a free `.dweb` domain derived from the project name
- **Domain persistence** вҖ” Domains survive restarts via `/tmp/dweb-domains.json`
- **P2P content proxy** вҖ” Browse content from peer instances' `.dweb` domains via the built-in proxy endpoint, avoiding CORS issues

### рҹӨ– AI Build Agent with 15+ Providers
Generate full-stack applications from natural language:
- **15+ AI providers**: Ollama, NVIDIA NIM, Groq, Gemini, DeepSeek, Mistral, OpenAI, Anthropic, Together, OpenRouter, HuggingFace, Fireworks, Cohere, Cerebras, xAI, Hyperbolic
- **100+ free models** вҖ” No API key needed for most providers
- **Offline-capable** вҖ” Ollama runs 100% locally
- **OpenCode CLI integration** вҖ” Full agentic coding workflow

### рҹ“Ғ P2P File Sharing
Drag-and-drop file sharing between dweb instances:
- Upload files via browser
- Share directly to P2P peers
- Download shared files from any instance
- File discovery across the P2P network

### рҹ”§ Built-in Browser with dweb Protocol
Full browser tab with `dweb://` protocol support:
- Browse the web within dweb
- Tutorials for building sites, APIs, and PHP apps
- Bookmark manager
- Multiple search engines

## P2P Connection & Discovery UI

dweb provides a rich in-browser interface for managing P2P connections, all accessible from the **Dashboard's Network section**.

### Network Status Bar

A real-time status bar shows your current networking state at a glance:

```
вқ¶ Mode: P2P Visible  |  Peers: 7  |  Relay: Connected  |  Signals: 3
```

- **Mode indicator** вҖ” Hover to see what each mode does (Local Only / P2P Visible / P2P Anonymous)
- **Peer count** вҖ” Number of actively connected peers; hover for summary
- **Relay status** вҖ” Connected/Offline with relay address on hover
- **Signal count** вҖ” Incoming WebRTC signaling requests (purple highlight)

### Online Mode Toggle

Three buttons toggle your instance's visibility:

| Mode | Effect |
|------|--------|
| **Local** | No P2P вҖ” services are local-only |
| **Visible** | Your instance is discoverable by other peers and can connect out |
| **Anonymous** | You can connect to peers but they cannot discover you |

Switching modes is persisted to `localStorage` across tab switches.

### Connect Dialog

Click the **Connect** button to open the connection modal with two modes:

#### Direct P2P
Enter `IP:Port` to connect directly вҖ” no relay needed. Works on LAN or when the remote peer has a public IP. Lower latency, no central dependency. May fail behind strict NATs or firewalls. Hover over the "Direct P2P" button for a detailed explanation panel.

#### Via Relay
Browse discovered peers from the relay, search/filter by ID or hostname, or enter a peer ID manually. The relay handles signaling/ICE/STUN for NAT traversal вҖ” data flows P2P once connected. Hover over the "Via Relay" button for a detailed explanation panel.

Both modes support:
- Manual address/peer ID input
- Optional display label
- Inline discovered peer browser with search filtering
- Relay-connected status indicator
- Warning banner when relay is offline

### Paginated Peer Lists

- **Discovered Peers** вҖ” All peers visible through the relay, displayed **15 per page** with Prev/Next navigation and a page counter (e.g. "Page 2 of 4"). Only shows pagination controls when there are multiple pages.
- **Connected Remotes** вҖ” Your connected remote instances, also **15 per page** with full navigation. Each entry shows status dot (green/yellow/red), name, mode badge (Visible/Anonymous/Relay), address, latency, and service list. Use the reconnect/disconnect/remove buttons per entry.

### Peer Persistence & Auto-Registration

- The local instance **auto-registers** as a peer on startup вҖ” the P2P tab always shows at least one peer
- Manual peer registrations and auto-discovered peers are **saved to disk** (`/tmp/dweb-peers.json`) and restored after server restarts
- Peers that go silent for 60 seconds are **automatically cleaned up**

## Cross-Peer .dweb Domain Resolution

When two or more dweb instances are connected via P2P, `.dweb` domains registered on any peer are **resolvable from any other peer**.

### How It Works

1. **Domain Registration** вҖ” Register a `.dweb` domain (e.g. `my-project.dweb`) via the Domains UI or `POST /api/domain/register`
2. **Auto-Domain Assignment** вҖ” Publishing a service with `auto_domain: true` automatically creates a domain matching the project name
3. **Local Resolution** вҖ” `GET /api/domain/resolve/:name` checks the local domain registry first
4. **Cross-Peer Fallback** вҖ” If the domain isn't found locally, dweb queries **all connected peers in parallel** via their `/api/domain/query/:name` endpoints using `Promise.allSettled` вҖ” results are cached for 60 seconds
5. **Content Proxy** вҖ” The resolved address is fetched through `/api/proxy/fetch?url=` to avoid CORS issues when rendering in the BrowserView

### BrowserView Integration

The built-in browser tab supports `dweb://` URLs:
- **Tauri mode**: Uses Rust IPC to invoke `resolve_domain` natively
- **Browser/Web IDE fallback**: Falls back to `fetch('/api/domain/resolve/:name')` and proxies content through the dweb server вҖ” works in any browser

### Architecture

## Best Use Cases

### 1. Personal Cloud Development Environment
Replace Docker Compose, ngrok, and Heroku with a single install. Start/stop services, view logs, and manage ports вҖ” all from your browser.

### 2. AI-Powered Code Generation
Use the built-in AI Build Agent to scaffold full-stack apps from natural language. With Ollama running locally, it works **completely offline**.

### 3. P2P Service Sharing
Host a service on your dweb node and share it directly with other dweb users across the P2P network. No central server, no CDN, no cloud bill.

### 4. Multi-Instance Collaboration
Run multiple dweb instances on different machines. Each instance discovers the others via P2P relay, and services are accessible across instances.

### 5. P2P File Sharing
Drag-and-drop file sharing between dweb instances. Files are transferred directly P2P, stored locally on the receiving instance.

### 6. Self-Hosted Websites & Portfolios
Deploy static sites, blogs, and portfolios on your own machine with a `.dweb` domain.

### 7. Offline-First Development
With Ollama running locally, the AI Build Agent works without internet. Perfect for air-gapped environments or privacy-conscious workflows.
